Ubik processes all payments through [Stripe](https://stripe.com). The backend manages secret keys and payment intent creation; the frontend loads the Stripe SDK dynamically and mounts Stripe Elements for PCI-compliant card capture.

## Payment Flow

<Steps>
  <Step title="Client selects room and time window">
    The user picks a date, entry time, and exit time inside the `PaymentModal`. Total price is computed as `room.price × totalHours` (in COP).
  </Step>

  <Step title="Create reservation">
    The frontend calls `POST /reservations` to create a reservation record before any payment is attempted.

    ```typescript
    // PaymentService.createReservation()
    this.http.post<{ id: number; confirmationCode?: string }>(
      `${environment.apiUrl}/reservations`,
      { roomId, motelId, totalPrice, userId, checkInDate, checkOutDate }
    );
    ```

    The response includes the new `reservation.id` and an optional `confirmationCode`.
  </Step>

  <Step title="Fetch Stripe publishable key">
    On modal initialisation, the app fetches the Stripe publishable key from the backend:

    ```typescript
    // PaymentService.getStripeConfig()
    this.http.get<{ publishableKey: string }>(
      `${environment.apiUrl}/payments/config`
    );
    ```

    The key is used to load the Stripe.js SDK via `loadStripe(config.publishableKey)`.

    <Warning>
      The publishable key is the **only** Stripe credential that ever reaches the browser. The secret key stays on the backend at all times.
    </Warning>
  </Step>

  <Step title="Create Payment Intent">
    With the reservation ID in hand, the frontend requests a Payment Intent:

    ```typescript
    // PaymentService.createPaymentIntent()
    this.http.post<{ paymentId: number; clientSecret: string }>(
      `${environment.apiUrl}/payments/create-intent`,
      { reservationId, amountCop }
    );
    ```

    The backend creates the Stripe Payment Intent, stores it, and returns the `clientSecret` and an internal `paymentId`.
  </Step>

  <Step title="Mount Stripe Elements">
    The `clientSecret` is passed to `stripe.elements()` and a `payment` element is mounted in the modal DOM:

    ```typescript
    this.elements = this.stripe!.elements({
      clientSecret: this.clientSecret!,
      appearance: {
        theme: 'night',
        variables: {
          colorPrimary: '#A72027',
          colorBackground: '#1a1a1f',
          colorText: '#ffffff',
          colorDanger: '#ff4d4f',
          fontFamily: 'DM Sans, sans-serif',
          borderRadius: '8px',
        },
      },
    });

    const paymentElement = this.elements.create('payment');
    paymentElement.mount('#payment-element-desktop'); // or '#payment-element-mobile'
    ```

    Stripe Elements renders a secure, hosted card input — no raw card numbers pass through the Ubik application.
  </Step>

  <Step title="Confirm payment">
    When the client clicks **Confirm Payment**, the frontend calls `stripe.confirmPayment()`:

    ```typescript
    const { error } = await this.stripe.confirmPayment({
      elements: this.elements,
      confirmParams: {
        return_url: window.location.origin + '/reservations/success',
      },
      redirect: 'if_required',
    });
    ```

    - `redirect: 'if_required'` means most payments complete without a full page redirect (e.g. standard card payments). 3D Secure or redirect-based methods will use the `return_url`.
  </Step>

  <Step title="Handle success or failure">
    - **Success**: the modal closes and the user is navigated to `/payment/success` with payment details in router state.
    - **Failure**: the Stripe error message is displayed inline; the modal stays open so the user can retry.
  </Step>
</Steps>

## Service Interface

```typescript
export interface StripeConfig {
  publishableKey: string;
}

export interface CreatePaymentResponse {
  paymentId: number;
  clientSecret: string;
}

@Injectable({ providedIn: 'root' })
export class PaymentService {
  // Create a reservation before initiating payment
  createReservation(
    roomId: number,
    motelId: number,
    totalPrice: number,
    userId: number,
    checkInDate: string,   // ISO datetime string
    checkOutDate: string,  // ISO datetime string
  ): Observable<{ id: number; confirmationCode?: string }>

  // Fetch Stripe publishable key from backend
  getStripeConfig(): Observable<StripeConfig>

  // Create a Stripe Payment Intent
  createPaymentIntent(
    reservationId: number,
    amountCop: number,
  ): Observable<CreatePaymentResponse>
}
```

## Payment Outcome Routes

<Tabs>
  <Tab title="Success — /payment/success">
    Displayed after a successful `stripe.confirmPayment()` call. The page receives payment details via Angular router state (`navigation.extras.state`).

    **Shown to the user:**
    - Confirmation badge ("Confirmada")
    - Total paid in COP
    - Check-in date and time window
    - Room / motel name
    - QR code (`UBIK-{sequenceNumber}`) for express check-in
    - Full confirmation code for manual entry at reception

    The page requires no API call — all data is passed from the modal.
  </Tab>
  <Tab title="Failure — /payment/failure">
    Shown when the payment is definitively declined (e.g. card rejected by Stripe, not just a modal-level error).

    **Shown to the user:**
    - Error icon
    - Message: "No charge has been made to your card"
    - Button to retry (`/explore`)
    - Button to return home (`/`)
  </Tab>
</Tabs>

## Security Notes

<CardGroup cols={2}>
  <Card title="No raw card data" icon="shield-check">
    All card entry is handled by Stripe Elements, hosted iframes served from Stripe's domain. Raw PAN, CVV, and expiry never touch Ubik servers or client-side application code.
  </Card>
  <Card title="Secret key server-side only" icon="key">
    The Stripe secret key lives exclusively on the backend. The frontend only ever receives the publishable key (via `GET /payments/config`) and a `clientSecret` scoped to a single Payment Intent.
  </Card>
  <Card title="Interceptor bypass for Cloudinary" icon="lock">
    The `CloudinaryService` uses `HttpBackend` directly to bypass Angular's HTTP interceptors, ensuring the JWT token is never accidentally sent to Cloudinary.
  </Card>
  <Card title="Auth guard on reserve" icon="user-lock">
    If a user tries to complete a booking without being logged in, the modal closes and they are redirected to `/login`. The room selection is not lost.
  </Card>
</CardGroup>

## Currency

All amounts are denominated in **Colombian Pesos (COP)**. The `amountCop` field passed to `createPaymentIntent` is the raw integer value in COP. The Stripe Payment Intent is created in COP on the backend.
