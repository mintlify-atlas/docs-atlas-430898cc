## Prerequisites

Before installing Ubik, ensure your development environment meets the following requirements.

| Requirement | Minimum version | Notes |
|---|---|---|
| Node.js | 18.x | LTS recommended; required by Angular 20 |
| pnpm | 10.x | Specified in `package.json` `packageManager` field |
| Angular CLI | 20.x | Install globally via npm |
| Git | Any recent version | For cloning the repository |

Install the Angular CLI globally if you haven't already:

```bash
npm install -g @angular/cli
```

Install pnpm if you prefer it over npm:

```bash
npm install -g pnpm
```

<Tip>
  Run `node -v` and `ng version` to verify your installed versions before proceeding.
</Tip>

## Clone the repository

```bash
git clone https://github.com/Juankos0714/Ubik-App.git
cd Ubik-App
```

## Install dependencies

<CodeGroup>
```bash pnpm
pnpm install
```

```bash npm
npm install
```
</CodeGroup>

<Note>
  The repository ships with a `pnpm-lock.yaml` lockfile. Using pnpm ensures deterministic installs that match CI and other contributors' environments.
</Note>

## Environment configuration

Ubik uses Angular's built-in environment file system. The development configuration lives at `src/environments/environment.ts` and is replaced with `src/environments/environment.prod.ts` during production builds.

```typescript
// src/environments/environment.ts
export const environment = {
  production: false,
  apiUrl: 'https://ubik-back.duckdns.org/api',
  authUrl: 'https://ubik-back.duckdns.org/auth',
  cloudinary: {
    cloudName: 'du4tcug9q',
    uploadPreset: 'angular_images',
    folder: 'Ubik',
  },
};
```

### Environment variables explained

| Variable | Description |
|---|---|
| `production` | Set to `true` in the production file; controls Angular optimizations |
| `apiUrl` | Base URL for all REST API calls (motels, rooms, reservations, users) |
| `authUrl` | Base URL for authentication endpoints (login, register, token refresh) |
| `cloudinary.cloudName` | Cloudinary account identifier for image uploads |
| `cloudinary.uploadPreset` | Cloudinary unsigned upload preset name |
| `cloudinary.folder` | Destination folder in Cloudinary for all uploaded assets |

<Warning>
  Do not commit credentials or private API keys to the environment files. If you are self-hosting the backend, update `apiUrl` and `authUrl` to point to your own instance before building.
</Warning>

## Build commands

### Development server

Starts a local server at `http://localhost:4200` with hot-module replacement:

<CodeGroup>
```bash pnpm
pnpm start
# or
npx ng serve
```

```bash npm
npm start
```
</CodeGroup>

### Production build

Compiles and bundles the application with full optimizations. The Angular CLI automatically swaps `environment.ts` for `environment.prod.ts`:

<CodeGroup>
```bash pnpm
pnpm run build
```

```bash npm
npm run build
```
</CodeGroup>

Output is written to `dist/frontend/`.

### Watch mode (development)

Rebuilds on every file change without starting a dev server — useful when you are running the SSR server separately:

```bash
npm run watch
# equivalent to: ng build --watch --configuration development
```

### Server-side rendering

Ubik ships with Angular Universal + Express for SSR. After a production build, start the SSR server with:

```bash
npm run serve:ssr:frontend
# equivalent to: node dist/frontend/server/server.mjs
```

The SSR entry point is configured in `angular.json` under `architect.build.options.ssr.entry` pointing to `src/server.ts`.

### Unit tests

```bash
npm test
# equivalent to: ng test
```

Tests run in Chrome via Karma and Jasmine.

## User roles and role IDs

Ubik's permission system uses numeric role IDs to gate routes and UI features. These are fixed values defined in the backend:

| Role | Role ID |
|---|---|
| Admin | `7392841056473829` |
| Owner | `3847261094857362` |
| Client | `9182736450192837` |

Route-level guards (e.g. `permissionGuard('view_dashboard')` in `app.routes.ts`) check the authenticated user's role before allowing navigation to `/dashboard/admin` or `/dashboard/owner`.

## Troubleshooting

<AccordionGroup>
  <Accordion title="`ng: command not found` after installing Angular CLI">
    The Angular CLI binary may not be on your PATH. Try running it via npx:

    ```bash
    npx ng serve
    ```

    Or reinstall globally and restart your terminal:

    ```bash
    npm install -g @angular/cli
    ```
  </Accordion>

  <Accordion title="`pnpm install` fails with a lockfile compatibility error">
    Your local pnpm version may not match the one specified in `package.json`. Update pnpm to the required version:

    ```bash
    npm install -g pnpm@10
    ```

    Then re-run `pnpm install`.
  </Accordion>

  <Accordion title="App starts but API calls return CORS or network errors">
    Verify that `apiUrl` and `authUrl` in `src/environments/environment.ts` are reachable from your machine. The hosted backend at `ubik-back.duckdns.org` may be unavailable if you are running a local backend instance. Update the URLs to match your backend address.
  </Accordion>

  <Accordion title="Build fails with budget exceeded errors">
    The production build enforces size budgets defined in `angular.json` (500 kB warning / 1 MB error for the initial bundle). If your build exceeds the limit:

    - Check for unintentionally large third-party imports.
    - Use Angular's `--stats-json` flag and `webpack-bundle-analyzer` to identify heavy modules.
    - Consider lazy-loading feature routes.
  </Accordion>

  <Accordion title="SSR server fails to start after `npm run serve:ssr:frontend`">
    The SSR server requires a completed production build. Run `npm run build` first, then start the server:

    ```bash
    npm run build && npm run serve:ssr:frontend
    ```

    Ensure `dist/frontend/server/server.mjs` exists before running the serve command.
  </Accordion>

  <Accordion title="Maps not rendering (blank Leaflet tiles)">
    Leaflet's CSS must be loaded for tiles to display correctly. Confirm that `node_modules/leaflet/dist/leaflet.css` is listed in the `styles` array of `angular.json` — it is included by default in the project configuration.
  </Accordion>
</AccordionGroup>
