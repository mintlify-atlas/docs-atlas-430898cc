<Note>
  This guide assumes you have **Node.js 18+**, **pnpm**, and the **Angular CLI** installed. See [Installation](/installation) for detailed prerequisites.
</Note>

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Juankos0714/Ubik-App.git
    cd Ubik-App
    ```
  </Step>

  <Step title="Install dependencies">
    The project uses pnpm as its package manager (defined in `package.json` `packageManager` field).

    ```bash
    pnpm install
    ```

    <Tip>npm and yarn also work, but pnpm is recommended to stay consistent with the lockfile.</Tip>
  </Step>

  <Step title="Configure environment variables">
    The app reads its backend URLs and Cloudinary settings from `src/environments/environment.ts`. The defaults point to the hosted API and are ready to use out of the box:

    ```typescript
    // src/environments/environment.ts
    export const environment = {
      production: false,
      apiUrl: 'https://ubik-back.duckdns.org/api',
      authUrl: 'https://ubik-back.duckdns.org/auth',
      cloudinary: {
        cloudName: 'du4tcug9q',
        uploadPreset: 'angular_images',
        folder: 'Ubik',
      },
    };
    ```

    <Warning>
      If you are running your own backend instance, update `apiUrl` and `authUrl` before starting the dev server. For a production build the Angular CLI swaps this file for `src/environments/environment.prod.ts`.
    </Warning>
  </Step>

  <Step title="Start the development server">
    ```bash
    npm start
    # equivalent to: ng serve
    ```

    Angular CLI compiles the app and starts a local dev server with hot-module replacement.
  </Step>

  <Step title="Open the app in your browser">
    Navigate to:

    ```
    http://localhost:4200
    ```

    You should see the Ubik home page. From here you can explore motels on the map, register as a client or motel owner, and log in.
  </Step>
</Steps>

## Available npm scripts

| Command | Description |
|---|---|
| `npm start` | Start the development server (`ng serve`) |
| `npm run build` | Production build |
| `npm run watch` | Build in watch mode (development config) |
| `npm test` | Run unit tests via Karma |
| `npm run serve:ssr:frontend` | Start the SSR Express server after a production build |
